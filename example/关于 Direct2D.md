# 关于 Direct2D

**项目 • 2023/06/13**

本主题介绍 Direct2D，这是一种 API，它为 Win32 开发人员提供以卓越的性能和视觉质量执行二维图形呈现任务的能力。

Direct2D 是一种硬件加速的即时模式二维图形 API，可为 2D 几何图形、位图和文本提供高性能和高质量的渲染。Direct2D API 旨在与使用 GDI、GDI+ 或 Direct3D 的现有代码进行互操作。

## 什么是 Direct2D？
Direct2D 是一种硬件加速的即时模式二维图形 API，可为 2D 几何图形、位图和文本提供
高性能和高质量的渲染。 Direct2D API 旨在与使用 GDI、GDI+ 或 Direct3D 的现有代码
进行互操作。
Direct2D 主要供以下类开发人员使用：
大型企业级本机应用程序的开发人员。
创建控件工具包和库供下游开发人员使用的开发人员。
需要服务器端呈现二维图形的开发人员。
使用 Direct3D 图形的开发人员需要简单、高性能的 2D 和文本呈现，用于菜单、用
户界面 (UI) 元素，以及抬头显示 (HUD) 。

## 为什么选择 Direct2D？

在 Microsoft Windows 中创建新的 2-D 图形 API 的主要动机包括：

-   跟上 Windows 用户习惯的视觉丰富度不断提升的步伐。
-   使开发人员能够编写与运行它的电脑的图形处理硬件直接缩放的二维呈现代码。
-   使开发人员能够编写代码来呈现可在服务上下文中运行的二维图形。

近年来，最终用户开始期待数字体验中更高的视觉保真度。这种趋势反映在消费电子产品（GPS 设备、媒体播放设备、移动电话和数码相机）以及电影、电视、视频游戏和 Web 图形内容的多样性中。为了跟上这些变化，始终要求开发人员将其现有 Windows 应用程序提升到更丰富的视觉对象级别。

现代 Windows 电脑中的图形处理器也在稳步发展。虽然 Direct3D 服务于高端 3D 图形应用程序，WPF 满足 .NET 开发人员的需求，但对于拥有基于 GDI/GDI+ 的大量现有代码库，或想要在 Direct3D 应用程序中合并高质量 2D 图形的开发人员来说，存在差距。此外，对可在服务中使用的图形 API 的需求已成为企业和 Web 开发中的新兴要求，现有 API 在服务上下文中使用时可能在可靠性和可伸缩性方面存在不足。

## 什么是 Direct2D？

### 具有最大可用性的高性能

Direct2D 是使用 Direct3D 10.1 API 生成的用户模式库（*注：从 Windows 8 开始，Direct2D 是使用 Direct3D 11.1 API 生成的*）。这意味着 Direct2D 应用程序受益于现代主流 GPU 上的硬件加速渲染。硬件加速也通过使用 Direct3D 10 级别 9 渲染在早期 Direct3D 9 硬件上实现。

-   **软件回退：** 对于无法使用硬件加速的情况，Direct2D 包括高性能软件光栅器。在软件中呈现时，性能明显优于 GDI+ 且具有类似的视觉质量。软件光栅器还设计用于服务上下文。
-   **远程桌面支持：** 可以使用 Windows 7 操作系统中的远程桌面协议 (RDP) 基础结构远程显示内容。开发人员可以选择由显示计算机的 GPU 处理呈现，还是在本地呈现并作为位图传输。

通过提供单个 API，结合 Direct3D 的性能、软件回退、远程桌面和服务渲染，Direct2D 使开发人员能够在许多不同的方案中实现高性能呈现。

### 视觉质量

使用 Direct2D 的应用程序可以提供比 GDI 更高的视觉质量：

-   **抗锯齿：** 使用按基元抗锯齿，提供更平滑的曲线和线条。
-   **透明度：** 完全支持透明度和 alpha 混合。
-   **别名呈现：** 开发人员可以指定矢量图形的别名呈现，用于需要贴靠到硬像素边界的方案（如指针或标尺等 UI 元素），或匹配 GDI 样式输出。

### 互操作性

通过与 GDI 和 Direct3D 的图面级互操作性，开发人员可以更轻松地集成基于 Direct2D 的渲染：

-   主要使用 GDI、GDI+ 或 Direct3D 的应用程序可以逐步迁移到 Direct2D。
-   便于使用 DirectWrite 获取高质量文本。
-   便于使用 Microsoft Windows 图像处理组件 (WIC) 的高级图像处理功能。

## 总结

Microsoft Direct2D 使开发人员能够在其应用程序中生成二维图形功能，提供比 GDI 更高的视觉质量，以及使用新式 GPU 进行缩放的性能特征。借助 Direct2D 互操作性模型，开发人员可以有选择地一次迁移其应用程序的某些部分以及基于 GDI、GDI+ 或 Direct3D 的呈现。
